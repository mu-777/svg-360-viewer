Why bother rendering 360° panoramas as SVG? 🌀

View on GitHub

Grab the panorama below and look around — scroll or pinch to zoom. 👀 No WebGL. No Three.js. No 3D library at all. It's just HTML and a single SVG filter (feImage + feDisplacementMap). The 360° projection is pure declarative SVG — the only JavaScript is a few lines for drag & zoom.

↔ drag  ·  🔍 scroll / pinch to zoom

“Wait, that's the whole thing?” 🤯

Pretty much. The viewer up top is just a displacement-map filter, your panorama dropped into a <pattern>, and one line of JS for the drag:

It's just a DOM element 🧩

Each viewer is a plain SVG element, not a WebGL canvas — so you're not fighting the browser's context limit. Browsers keep only ~16 live WebGL contexts at a time (source), which caps how many WebGL 360° viewers fit on one page. SVG doesn't care — the grid below is running 20 of them at once, comfortably past that ceiling. 😎
